The First Principle

How everything works

You don't write code. You connect components using UnityEvents. Every interaction — from opening a door to tracking a score — follows the same structure.

Component Reference

Every component, one line

Each one is a trigger (fires an event) or a function target (responds to one).

Input fires an event → Event calls a function → Function does something

Every interaction in EGTK follows this structure, no matter how complex the result.

First Principle Check
1. What happens? Pick an Input.
2. What responds? Pick a function.
3. Wire them in the Inspector.
